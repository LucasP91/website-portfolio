# Sources & Citations

This skill's material is distilled from these open, public sources. Cite the relevant one when making a non-obvious recommendation.

## Primary sources

1. **W3C — Web Platform Design Principles**
   <https://www.w3.org/TR/design-principles/> (a.k.a. w3.org/design-principles), W3C Group Note.
   *Used for:* the user-first ethos and device independence — "Put user needs first" (Priority of Constituencies: users › authors › implementors), "Support the full range of devices and platforms" (media independence), "Minimize user data," "It should be safe to visit a web page." Grounds the accessibility, responsive, and privacy stance throughout.

2. **robinstickel/awesome-design-principles**
   <https://github.com/robinstickel/awesome-design-principles>
   *A curated list of good design and experience principles* — defines principles as "high level rules that guide detailed design decisions," and indexes real-world design systems (Apple HIG, Google Material, IBM Carbon, Shopify Polaris, etc.) and articles. *Used for:* the design-system / consistency mindset and pointers to authoritative system docs.

3. **nicolesaidy/awesome-web-design**
   <https://github.com/nicolesaidy/awesome-web-design>
   Curated web-design resources organized by Inspiration, Colors, Typography, Icons, Images, and Guidelines (incl. iOS Human Interface Guidelines, Google Material Design). *Used for:* color, typography, and platform-guideline references.

4. **bradtraversy/design-resources-for-developers**
   <https://github.com/bradtraversy/design-resources-for-developers>
   Developer-focused resource list: fonts, color tools, icons, CSS frameworks & methodologies (BEM/OOCSS), CSS animations, JS animation/chart libraries, UI component kits. *Used for:* practical tooling cited in the color, typography, and motion notes.

## Standards referenced (via the above)

- **WCAG 2.2** (W3C Web Content Accessibility Guidelines) — <https://www.w3.org/TR/WCAG22/> — contrast, keyboard, focus, target size, reduced motion.
- **WAI-ARIA Authoring Practices** — <https://www.w3.org/WAI/ARIA/apg/> — accessible component patterns.

## Citation style for this skill

When you apply a rule, attribute it briefly, e.g.:
- "Per WCAG 2.2 (1.4.3), body text needs ≥ 4.5:1 contrast."
- "Following the W3C 'media independence' principle, this layout is fluid from 320px up."
- "Per the design-system consistency principle (awesome-design-principles), reuse the spacing tokens rather than ad-hoc margins."
